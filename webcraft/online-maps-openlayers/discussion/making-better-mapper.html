Making a better mapper

Nick Doiron on Tue, 2011-03-22 22:24
Go back to: General discussion

We have only four days left of the course, and I'd like to end it with an open discussion to help ourselves and future visitors to the course page to become better mappers.  That includes acknowledging the power of non-open systems like Google Maps, and sharing links about HTML or Firebug.

A couple of things which really impress me about non-open maps are

  • The Birdseye view in Bing Maps. The detail and perspective from their photos are amazing.
  • Adding any KML and Fusion Tables to Google Maps

I wanted to re-post some links which I'd post on the first day if we had this course again:

  • OpenStreetMap Beginners Guide shared by trekr5
  • Video of tracing a building in Potlatch 2, created by berniejconnors
  • OpenGeo's OpenLayers tutorial (helpful, but outdated in places)
  • Grassroots mappers use kites and balloons to make their own aerial maps
  • OpenLayers official examples - and documentation

I remember someone's helpful JavaScript reference but I can't find it.  If you can re-post the link here, please do.

Timothy Barmann at the Providence Journal posted an excellent tutorial on mapping with QGIS, JavaScript and jQuery http://www.projo.com/temp/tim/nicar/js_mapping/tutorial/ as well as an introduction to QGIS http://www.projo.com/temp/tim/nicar/qgis_tutorial/#0

For those who are primarily interested in publishing maps that show information -- that join data with a geographic component (population growth, poverty levels, etc.) to a map -- another interesting project is OpenHeatMap. You can't actually create your own map polygons, but it does support a wide variety of location input types. http://www.openheatmap.com

Note sure what JavaScript tutorial you were referring to, but I did see one at W3schools: http://www.w3schools.com/js/default.asp
